## [3.1.0] - 2025-01-12

### Added ✨
- **빠른 필터 시스템**: "문제 있는 서비스만 보기" 원클릭 토글 필터
- **즐겨찾기 빠른 필터**: "즐겨찾기만 보기" 원클릭 토글 필터  
- **키보드 단축키 시스템**: R(새로고침), F(필터), Esc(닫기) 지원
- **이미지 Lazy Loading**: `loading="lazy"`, `decoding="async"` 적용
- **Critical 이미지 Preload**: 주요 6개 서비스 아이콘 우선 로드
- **에러 메시지 시스템**: 서비스별 사용자 친화적 에러 메시지 표시
- **모바일 스크롤 인디케이터**: 상/하단 그라데이션으로 스크롤 가능 영역 표시

### Improved 🎯
- **모바일 터치 스크롤**: iOS Safari 네이티브 스크롤 지원 (`-webkit-overflow-scrolling: touch`)
- **에러 처리 로직**: 지수 백오프 재시도 메커니즘 구현
- **중복 요청 방지**: `pendingRequests` + `isGlobalRefreshing` 상태 관리
- **필터링 성능**: 기존 컴포넌트 필터와 빠른 필터의 효율적인 조합
- **모바일 모달 UX**: 터치 최적화 + 커스텀 스크롤바 스타일링

### Fixed 🐛
- **모바일 필터 모달 스크롤**: iOS Safari에서 부드러운 스크롤 경험
- **API 응답 실패**: fallback 로직 및 재시도 메커니즘 강화
- **동시 API 호출**: 중복 요청 방지로 안정성 개선
- **에러 표시**: 서비스별 개별 에러 상태 관리

### Technical 🛠️
- **Zustand 스토어 확장**: `quickFilters`, `serviceErrors` 상태 추가
- **ErrorHandler 클래스**: 에러 분류 및 사용자 메시지 생성 로직
- **CSS 모듈화**: 모바일 터치 최적화 스타일 추가
- **타입 안전성**: 모든 신규 기능에 TypeScript 타입 적용
- **성능 최적화**: 이미지 로딩 전략 개선

### Localization 🌍
- **키보드 단축키 안내**: 한국어/영어 번역 추가
- **빠른 필터 라벨**: "문제만", "즐겨찾기만" / "Issues Only", "Favorites Only"

## [3.0.0] - 2025-01-01

### Added ✨
- **모듈화된 컴포넌트 아키텍처**: 1814줄 단일 컴포넌트 → 15개+ 전문화된 컴포넌트
- **Zustand 상태 관리**: 20개+ useState → 중앙화된 스토어 + 영속성 지원
- **반응형 디자인 시스템**: 통합된 브레이크포인트 + 컴포넌트 기반 조건부 렌더링
- **Tailwind CSS 4.1 완전 통합**: CSS 변수 + 타입 안전성 + 유틸리티 함수
- **다국어 JSON 시스템**: 네임스페이스 기반 번역 + 템플릿 변수 지원
- **접근성 완전 지원**: WCAG 가이드라인 준수 + 다국어 aria-label
- **CSS 모듈화**: 1488줄 단일 파일 → 8개 카테고리별 모듈

### Improved 🎯
- **성능 최적화**: React.memo + useMemo + useCallback 대폭 적용
- **개발자 경험**: TypeScript 타입 안전성 + VS Code Tasks + ESLint
- **빌드 시스템**: GitHub Actions + AWS ECR + Docker 멀티스테이지 빌드

### Removed ❌
- **레거시 컴포넌트**: 단일 거대 컴포넌트 구조
- **인라인 스타일**: CSS-in-JS 방식에서 모듈화된 CSS로 전환
- **하드코딩된 번역**: 하드코딩된 텍스트를 JSON 기반 시스템으로 대체

### Technical 🛠️
- **아키텍처 전면 개편**: 모놀리식 → 모듈화된 컴포넌트 시스템
- **상태 관리 통합**: 분산된 로컬 상태 → 중앙화된 Zustand 스토어
- **타입 시스템 강화**: 부분적 TypeScript → 100% 타입 안전성
- **빌드 최적화**: Vite 6.3 + 최신 플러그인 생태계

## 🏷️ 버전 정책

### Semantic Versioning
- **MAJOR (X.0.0)**: 호환되지 않는 API 변경
- **MINOR (0.X.0)**: 이전 버전과 호환되는 새로운 기능 추가
- **PATCH (0.0.X)**: 이전 버전과 호환되는 버그 수정

### 릴리스 스케줄
- **Major 릴리스**: 6개월마다 (주요 아키텍처 변경)
- **Minor 릴리스**: 1-2개월마다 (새로운 기능 추가)
- **Patch 릴리스**: 필요시 (긴급 버그 수정)

### 지원 정책
- **현재 버전**: 모든 지원 (버그 수정, 기능 추가, 보안 업데이트)
- **이전 메이저 버전**: 버그 수정 및 보안 업데이트만
- **구 버전**: 보안 업데이트만 (6개월간)
